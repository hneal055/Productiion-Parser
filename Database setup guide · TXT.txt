📊 DATABASE STRUCTURE
════════════════════════════════════════════════════════════════════════════════

TABLES:
───────────────────────────────────────────────────────────────────────────────

1. budget_analyses
   • Main storage for budget analysis results
   • Stores: filename, budget totals, risk scores, analysis data
   • One row = one uploaded budget file

2. budget_line_items
   • Individual line items from budgets
   • Enables detailed queries and filtering
   • Linked to budget_analyses

3. budget_comparisons
   • Stores comparison results
   • Links two analyses together
   • Saves comparison calculations

4. user_activity
   • Tracks all user actions
   • For analytics and debugging
   • Audit trail

5. app_settings
   • Application configuration
   • Key-value storage
   • User preferences

🔧 DATABASE UTILITIES
════════════════════════════════════════════════════════════════════════════════

VIEW STATISTICS:
───────────────────────────────────────────────────────────────────────────────
python database_utils.py stats

Shows:
  • Total analyses
  • Total budget tracked
  • Risk distribution
  • Recent activity


CREATE BACKUP:
───────────────────────────────────────────────────────────────────────────────
python database_utils.py backup

Creates timestamped backup in backups/ folder


CLEANUP OLD DATA:
───────────────────────────────────────────────────────────────────────────────
# See what would be deleted (90+ days old)
python database_utils.py cleanup 90

# Actually delete old data
python database_utils.py cleanup 90 --execute


SEARCH BUDGETS:
───────────────────────────────────────────────────────────────────────────────
python database_utils.py search "2024"

Finds all budgets with "2024" in filename


EXPORT TO CSV:
───────────────────────────────────────────────────────────────────────────────
python database_utils.py export budget_data.csv

Exports all analyses to CSV file


OPTIMIZE DATABASE:
───────────────────────────────────────────────────────────────────────────────
python database_utils.py vacuum

Reclaims space and rebuilds indexes


LIST BACKUPS:
───────────────────────────────────────────────────────────────────────────────
python database_utils.py list-backups

Shows all available backups


RESTORE FROM BACKUP:
───────────────────────────────────────────────────────────────────────────────
python database_utils.py restore backups/budget_analysis_20240101_120000.db

Restores database from backup file

🎯 ADVANTAGES OF DATABASE
════════════════════════════════════════════════════════════════════════════════

VS JSON CACHE:
───────────────────────────────────────────────────────────────────────────────

| Feature | JSON Cache | SQLite Database |
|---------|-----------|-----------------|
| **Data Persistence** | Lost on error | ✅ Always persisted |
| **Query Speed** | Slow (load all) | ✅ Fast (indexed) |
| **Storage Limit** | ~10-50 analyses | ✅ Unlimited |
| **Data Integrity** | ❌ Can corrupt | ✅ ACID compliant |
| **Search/Filter** | Manual | ✅ SQL queries |
| **Backups** | Manual copy | ✅ Built-in tools |
| **Multi-user** | ❌ Not safe | ✅ Concurrent access |
| **Relationships** | Manual | ✅ Foreign keys |
| **Reporting** | Limited | ✅ Advanced |

🆘 TROUBLESHOOTING
════════════════════════════════════════════════════════════════════════════════

ISSUE: "No module named 'flask_sqlalchemy'"
───────────────────────────────────────────────────────────────────────────────
FIX: pip install Flask-SQLAlchemy


ISSUE: Migration fails with KeyError
───────────────────────────────────────────────────────────────────────────────
FIX: 
  1. Your JSON cache might be corrupted
  2. Run: python migrate_to_database.py --no-backup
  3. Check error message for specific missing key
  4. May need to manually edit JSON


ISSUE: Database locked
───────────────────────────────────────────────────────────────────────────────
FIX:
  1. Close all instances of the Flask app
  2. Make sure no other process is accessing the database
  3. Restart


ISSUE: Want to go back to JSON
───────────────────────────────────────────────────────────────────────────────
FIX:
  1. Your original JSON was archived, not deleted
  2. Find: budget_cache_archived_YYYYMMDD.json
  3. Rename back to: budget_cache.json
  4. Use old Flask app version


ISSUE: Database is huge
───────────────────────────────────────────────────────────────────────────────
FIX:
  1. Clean up old data: python database_utils.py cleanup 90 --execute
  2. Optimize: python database_utils.py vacuum
  3. Create backup: python database_utils.py backup
